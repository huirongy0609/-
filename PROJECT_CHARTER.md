# Project Charter

Status: Draft
Version: v0.1.0
Date: 2026-07-12

## 项目名称

中国信托制物业发展平台

## 项目定位

本项目是信托制物业知识、标准、案例、GEO 运营、网站发布和平台工程的统一协作工程。

GitHub 是当前正式项目办公室和唯一正式事实来源。所有正式文件、任务、关键决策、版本变更和交接记录都应在本仓库或关联 GitHub 工作流中留痕。

## 当前阶段

协作体系建立期。

本阶段只建立项目治理、任务流转、交接、审核、决策和 GitHub 模板，不开发新平台功能。

## 项目成员

- 杨慧荣：项目总架构师。
- ChatGPT：项目总设计师与项目协调人。
- Work：内容、研究、标准和正式文档生产。
- Codex：平台工程、GitHub、自动化、数据结构、网站和部署实施。

## 项目边界

允许：

- 建立协作治理文件。
- 建立任务、交接、决策和审批规则。
- 建立 GitHub Issue 和 PR 模板。
- 建立标签和项目看板方案。
- 记录仓库审查结论。

禁止：

- 重写现有专业内容。
- 重构现有网站。
- 修改信托制物业定义。
- 自动发布内容。
- 删除旧文件。
- 合并重复文件。
- 擅自迁移仓库。
- 启动知识中心、小程序或 AI 问答新功能开发。

## 主仓库原则

当前工作区只发现一个 Git 仓库，远端为 `https://github.com/huirongy0609/-.git`，默认分支为 `main`。

在未发现更正式、更完整、权限更稳定的仓库前，本仓库建议作为“中国信托制物业发展平台”的主协作仓库。

## 成功标准

- 四类角色的职责、权限和禁止事项清晰。
- 任务可以从 Backlog 流转到 Completed。
- 每次任务完成有可追溯交接文件。
- 重大事项先形成决策记录。
- 已批准文件不被 Codex 直接修改正文。
- GitHub Issue、PR、Label、Project 方案可执行。
