# UI Principles

本文件定义《全国社区治理协同地图》的组件审美标准与界面准入规则。

## 1. 产品不是后台

界面不能像：

- 后台管理系统
- SaaS 管理台
- 传统 BI 大屏
- 表格型 OA
- 物业软件

界面必须像：

- 城市治理操作系统
- AI 协同中枢
- 数字孪生驾驶舱
- 国家级治理平台入口

## 2. 组件人格

所有组件必须具备：

- 克制
- 稳定
- 可信
- 低噪声
- 有空气感
- 有空间层级
- 有治理含义

## 3. 卡片原则

卡片不是信息堆叠容器，而是治理信息切片。

规则：

- 边框低对比。
- 背景半透明。
- 阴影轻。
- 圆角不超过 10px。
- 内边距稳定。
- 不做厚重浮层。

禁止：

- 白色卡片墙。
- 强阴影。
- 大圆角软萌风。
- 多层卡片套卡片。

## 4. 按钮原则

按钮表示治理动作。

主按钮：

- 使用 Civic Teal。
- 文案必须是行动级，如“进入全国地图”“提交治理线索”“加入城市共建”。

次按钮：

- 透明背景。
- 低对比边框。
- hover 只做轻微高亮。

禁止：

- 渐变按钮。
- 发光大按钮。
- 营销式 CTA。

## 5. 指标原则

指标必须像系统读数，不像营销数字。

规则：

- 数字可以大。
- 标签必须清楚。
- 必须有口径或状态说明。
- 趋势色必须克制。

禁止：

- 没有解释的巨大数字。
- 彩色指标墙。
- 夸张增长箭头。

## 6. 地图原则

地图是平台核心入口，不是装饰背景。

地图必须表达：

- 城市节点。
- 社区热区。
- 风险区域。
- 数据迁移。
- 协同连接。
- AI 扫描。

禁止：

- 装饰性 3D 地球。
- 无意义点阵。
- 满屏粒子。
- 红色警报热力图。

## 7. AI 模块原则

AI 模块必须像研判系统。

结构：

- 输入来源。
- 摘要状态。
- 关键证据。
- 风险判断。
- 关联案例。
- 建议动作。

禁止：

- 客服聊天窗口主导界面。
- 拟人头像。
- 魔法 AI 球。
- “AI 全自动解决”式文案。

## 8. 风险系统原则

风险视觉必须可研判，不制造恐慌。

规则：

- 中风险使用 Amber。
- 高风险少量使用 Restrained Red。
- 风险扩散是弱光晕。
- 风险卡片必须说明触发信号和影响范围。

禁止：

- 红色全屏。
- 警报灯。
- 高频闪烁。
- 危机电影感。

## 9. 文案原则

使用“AI 文明语言”。

推荐：

- 城市运行感知
- AI 协同网络
- 治理神经节点
- 风险扩散路径
- 政策解析队列
- 案例同步网络
- 共建信号流

禁止：

- 数据管理
- 系统监控
- 后台管理
- 功能模块
- 信息列表

## 10. 新组件准入清单

任何新组件进入系统前，必须确认：

- 没有传统后台味。
- 没有廉价科技感。
- 有空气感。
- 有深空感。
- 有未来治理感。
- 有国家级气质。
- 动效慢而克制。
- 信息真实可解释。
- 不引入不必要依赖。
- 不破坏现有视觉语言。
