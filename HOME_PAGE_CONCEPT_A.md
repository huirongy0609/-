# HOME_PAGE_CONCEPT_A

方案A：内容沉淀型首页

关键词：

```text
专业
可信
内容感
长期主义
```

参考：

```text
财新
得到
Notion
```

强调：

```text
内容
制度
案例
```

## 1. 首页首屏布局

首屏像一个高质量行业知识平台，而不是功能平台。

```text
顶部导航
聚道研究院 / 内容中心 / 制度中心 / 成长世界 / 产品中心 / 资源连接 / 治理实验室

--------------------------------------------------------------------------------

左侧主叙事

聚道研究院 出品

信托制社区治理平台

传播信托理念，沉淀治理经验，培养共治人才

这里持续整理信托制社区治理中的政策、案例、制度工具与实践经验，
帮助更多社区走向公开透明、共建共治与价值共享。

[ 查看最新实践 ]   [ 查找制度工具 ]

--------------------------------------------------------------------------------

右侧内容主视觉

一本被整理过的“治理经验册”
  - 政策动态卡片
  - 实践案例卡片
  - 制度模板卡片
  - 人物访谈摘录

底部轻入口

最新政策 / 实践案例 / 制度模板 / 深度观察
```

## 2. 首页整体结构

```text
第一屏：品牌与内容沉淀主视觉
第二屏：最近发生了什么
第三屏：值得关注的实践案例
第四屏：制度资源精选
第五屏：深度观察与人物访谈
第六屏：金牌管家成长世界入口
第七屏：产品中心入口
第八屏：用户识别与资源连接入口
第九屏：治理实验室样板间
```

## 3. 用户浏览路径

```text
进入首页
  ↓
看到平台长期沉淀的内容气质
  ↓
阅读最新政策或实践案例
  ↓
进入制度资源
  ↓
发现相关案例与工具
  ↓
进入成长世界或资源连接
```

用户的第一动作不是“注册”或“购买”，而是“阅读”和“理解”。

## 4. 视觉气质

整体气质：

```text
安静
克制
编辑感
知识感
行业可信感
```

视觉方式：

- 背景以暖白、浅灰、雾灰为主。
- 主色使用高级灰调青绿色，作为链接、标签、按钮和重点线条。
- 首屏主视觉采用内容纸张、制度文件、案例摘录、访谈卡片的层叠关系。
- 页面不使用强烈大图，不使用大屏科技纹理。
- 卡片像“编辑精选内容”，不是后台数据卡。

## 5. 为什么符合平台定位

平台定位是：

```text
传播信托理念，沉淀治理经验，培养共治人才
```

方案A最突出“沉淀治理经验”。

它让用户第一眼感受到：

- 这里有长期积累的内容。
- 这里有制度资源。
- 这里有真实案例。
- 这里不是临时搭建的功能系统。

它适合建立平台公信力，尤其适合面向社区、物业、业委会、研究者和合作机构。

## 6. 优点

- 最稳重，最可信。
- 内容平台气质强。
- 容易体现聚道研究院的专业背书。
- 适合长期运营。
- 不容易落入后台、驾驶舱或商城风格。
- 对政策、案例、观察、访谈等栏目承载能力强。

## 7. 风险

- 第一眼解决问题的行动感可能不够强。
- 成长世界和产品中心可能会被内容气质压住。
- 如果内容质量不足，页面会显得空。
- 容易被误解为行业媒体或知识库，而不是“内容 + 成长 + 资源连接平台”。

## 8. 禁止偏移

本方案禁止：

- 做成新闻门户。
- 做成传统 CMS。
- 做成政策文件列表。
- 做成文章堆砌。
- 做成政府网站。
