# Brand OS 压力测试报告

## 测试主题

为什么物业行业的问题不是服务问题，而是信任结构问题

## 测试目的

验证 Brand OS 是否能压住现有内容 Agent，不让输出滑回：

- 普通自媒体爆款模板
- 课程销售页
- SaaS 官网话术
- 培训机构招生文案
- 鸡血短视频口播

## 测试方式

本轮重点不是测试 API 连通，而是测试 Brand OS 对现有 Agent 的约束强度。

测试方法：

1. 使用同一主题。
2. 检查 `PromptLoader` 是否已向 Agent 注入 `brand_os/brand_os_compact.md`。
3. 对以下 Agent 做压力生成倾向审查：
   - `03_wechat_writer_agent.md`
   - `04_short_video_agent.md`
   - `05_live_selling_agent.md`
   - `06_xiaohongshu_agent.md`
   - `07_cover_prompt_agent.md`
4. 对照 Brand OS 检查是否失控。
5. 对高风险 Agent prompt 做局部回写，不新增 Agent，不扩展模块。

验证结果：

- Brand OS 注入：通过。
- Compact Brand OS 注入：通过。
- 风格训练库注入：通过。

# 1. 微信公众号长文 Agent

## 输出摘要

压力样本倾向：

- 标题会围绕“服务没有失效，失效的是信任”展开。
- 文章能使用七段结构。
- 核心论证能从“物业服务差”转向“账户、权责、公开、责任无法被共同确认”。
- 结尾容易自然引导到书籍、研究资料、项目诊断。

## 是否通过

修正后：通过。

## 失控点

原 prompt 中存在三个风险：

- “中文头部公众号”容易把文章拉回爆文模板。
- “有情绪、有转化”容易拉向营销号。
- “自然引导到书、课、咨询、社群或服务”容易变成课程销售页。

## 仍然像普通自媒体的句子

高风险表达：

> 文章要像中文头部公众号。

原因：这句话会激活爆款模板，而不是治理文明。

## 符合 Brand OS 的句子

推荐方向：

> 服务并没有失效。  
> 失效的是信任。

> 公开不是流程。  
> 公开本身就是治理。

## 已回写规则

- 把“头部公众号”收紧为“公共议题长文”。
- 把“书、课、咨询、社群或服务”收紧为“书籍、研究资料、项目诊断或治理体系入口”。
- 增加“纪录片旁白、治理档案、不得写成课程说明”的 Brand OS 写作边界。

## 需要修改的 Agent prompt

已修改：

- `prompts/03_wechat_writer_agent.md`

# 2. 短视频口播稿 Agent

## 输出摘要

压力样本倾向：

- 能生成 60 秒和 90 秒版本。
- 主题能从“物业服务差”转到“信任结构失效”。
- 如果没有约束，容易出现强钩子、强情绪、关注评论等平台化口播。

修正后的理想口播方向：

> 你以为物业行业最大的问题，是服务不好。  
> 但真正让小区失控的，不是服务。  
> 是信任没有地方存放。

## 是否通过

修正后：基本通过。

仍需在真实生成中继续观察。

## 失控点

原 prompt 中风险较高：

- “前 3 秒必须抓人”容易变营销号。
- “关注、评论、领取资料、进直播”容易变流量漏斗。
- “金句”容易变鸡血短句。

## 仍然像普通自媒体的句子

高风险表达：

> 前 3 秒必须抓人。

> 自然引导关注、评论、领取资料、看长文、进直播或了解服务。

原因：这会把口播拉回平台增长脚本。

## 符合 Brand OS 的句子

推荐方向：

> 钱没有被看见之前，信任不会自动发生。

> 物业矛盾不是突然爆发的。  
> 它是长期没有被记录的结果。

## 已回写规则

- 把“抓人”改为“停顿式判断”。
- 把“领取资料、进直播”收紧为“阅读、理解、查看书籍或项目诊断”。
- 增加“行业纪录片低声旁白”边界。
- 禁止鸡血、喊口号、课程销售页式收口。

## 需要修改的 Agent prompt

已修改：

- `prompts/04_short_video_agent.md`

# 3. 直播转化稿 Agent

## 输出摘要

压力样本倾向：

- 原 Agent 很容易进入“直播间成交脚本”。
- “卖书、卖课、服务”可以保留，但必须降级为理解入口，而不是成交压力。
- 直播内容应像一次公共议题说明会，而不是招生直播。

修正后的理想方向：

> 今天不急着讲方案。  
> 我们先把一个问题放到桌面上：  
> 为什么一个小区越服务，越不信任？

## 是否通过

修正后：勉强通过。

这是所有 Agent 里最高风险项。

## 失控点

原 prompt 中风险非常明显：

- Agent 名称叫 `Live Selling`。
- “卖课、训练营、社群或服务”容易变招生页。
- “逼单但不压迫”与 Brand OS 冲突。
- “成交意识”容易伤害公共性。

## 仍然像普通销售页的句子

高风险表达：

> 逼单但不压迫的话术。

> 这个产品正好能帮我少走弯路。

原因：这会直接破坏“治理文明”气质。

## 符合 Brand OS 的句子

推荐方向：

> 如果你愿意继续理解这套制度，可以从这本书开始。

> 项目诊断不是销售动作。  
> 它是把一个小区的信任结构先摊开。

## 已回写规则

- 将 Agent 名称从 “Live Selling” 改为 “Live Conversion / 直播克制转化”。
- 将“逼单但不压迫”改为“推进行动但不压迫”。
- 将默认转化从“课、训练营、陪跑服务”收紧为“书籍、研究资料、项目诊断或治理体系说明”。
- 明确禁止限时优惠、名额有限、立即报名、错过就晚了。

## 需要修改的 Agent prompt

已修改：

- `prompts/05_live_selling_agent.md`

后续建议：

- 如果未来继续做官网首页，不建议使用这个 Agent 的直播稿作为首页文案来源。
- 它只适合私域直播和深度说明场景。

# 4. 小红书文案 Agent

## 输出摘要

压力样本倾向：

- 原 prompt 会自然靠近“收藏、私信、生活化转折”的平台模板。
- 在 Brand OS 约束后，可以转向“治理困境的轻量表达”。
- 适合做行业观察短帖，不适合做流量引流帖。

修正后的理想方向：

> 很多人以为，小区矛盾是物业态度不好。  
> 但真正让人不安的是：  
> 这笔钱没有被共同看见。

## 是否通过

修正后：基本通过。

## 失控点

原 prompt 风险：

- “私信转化”容易变小红书引流帖。
- “搜索感和收藏感”容易触发干货模板。
- “我以前也这样”不适合物业治理议题，容易假装个人经历。

## 仍然像普通小红书模板的句子

高风险表达：

> 私信承接话术。

> 我以前也这样。

原因：会把公共治理议题写成个人成长帖或引流帖。

## 符合 Brand OS 的句子

推荐方向：

> 很多人卡住的不是服务体验。  
> 是不知道信任应该被放在哪里。

## 已回写规则

- 将“私信转化”改为“长期信任建立”。
- 将“私信承接话术”改为“深度交流承接话术”。
- 禁止“收藏这篇就够了”“私信领取”“保姆级教程”等平台模板。
- 将“我以前也这样”改为“很多人以为 / 真正卡住的是 / 需要被重新记录的是”。

## 需要修改的 Agent prompt

已修改：

- `prompts/06_xiaohongshu_agent.md`

# 5. 封面图提示词 Agent

## 输出摘要

压力样本倾向：

- 原 prompt 容易生成“冲突对比、移动端封面、强主体”的平台海报。
- Brand OS 需要它转向真实物件、公共空间、治理账本、档案室、低饱和光影。

修正后的理想画面方向：

> 清晨的社区档案室，一本亚麻封面的共同资产治理账本半开在旧木桌上，旁边是公开预算页、时间戳和磨砂玻璃反光，灰绿色低饱和，自然侧光，纪录片摄影，35mm，真实颗粒，大量留白。

## 是否通过

修正后：通过。

## 失控点

原 prompt 风险：

- “画面要有情绪张力”可能生成商业海报。
- “真实人物风 / 质感插画风 / 极简信息风”容易滑到通用封面模板。
- “一眼看出冲突”容易做成夸张营销图。

## 仍然像普通封面模板的句子

高风险表达：

> 真实人物风。

> 质感插画风。

> 极简信息风。

原因：这些是通用平台封面分类，不是品牌视觉系统。

## 符合 Brand OS 的句子

推荐方向：

> 纪录片真实空间风。

> 治理档案静物风。

> 克制信息秩序风。

## 已回写规则

- 明确封面要像可滚动的治理档案馆。
- 增加共同资产治理账本、公共空间、真实物件、纪录片摄影要求。
- 禁止 SaaS dashboard、培训招生海报、人物合影、握手签约、讲师站台。
- 将“冲突”改为“秩序变化”。

## 需要修改的 Agent prompt

已修改：

- `prompts/07_cover_prompt_agent.md`

# 6. Quality Check Agent 压力结论

## 原风险

`quality_check_agent` 原本仍然用“公众号爆文、传播力、爆款浓度、转化”等指标判断内容。

这些指标本身有用，但如果不加 Brand OS，会把内容推回：

- 爆文模板
- 转化模板
- 平台增长模板

## 已回写规则

已新增：

- Brand OS 一致性检查。
- 检查是否像治理文明，而不是课程产品、咨询产品或 SaaS。
- 检查是否有纪录片感、制度感、公共性。
- 出现课程销售页、SaaS 产品页、培训机构招生感或鸡血口播，必须 FAIL。

## 需要修改的 Agent prompt

已修改：

- `prompts/09_quality_check_agent.md`

# 7. 需要回写到 Brand OS 的规则

已回写：

1. 平台内容必须先像公共治理记录，再像平台内容。
2. 平台格式不能反过来吞掉品牌世界观。
3. 即使进入短视频，也不能鸡血。
4. 即使进入小红书，也不能私信引流化。
5. 即使进入直播，也不能招生逼单化。
6. 即使进入公众号，也不能营销号爆文化。
7. 禁止“私信领取、限时优惠、快速变现、爆款秘籍、狠狠拿捏、逼单”。

已修改文件：

- `brand_os/brand_os_compact.md`
- `brand_os/forbidden_rules.md`
- `brand_os/brand_voice.md`

# 8. 总体结论

## Brand OS 是否足够强

结论：基本足够，但直播转化 Agent 仍是最大风险点。

原因：

- 微信公众号、短视频、小红书、封面图在 Brand OS 约束后，已经能被压回“治理文明 / 纪录片 / 公共性”方向。
- 直播 Agent 天然带销售压力，即使改成克制转化，仍需要 quality_check 严格拦截。

## 当前通过情况

| Agent | 通过情况 | 风险等级 |
|---|---|---|
| 公众号长文 | 通过 | 中 |
| 短视频口播 | 基本通过 | 中 |
| 直播转化稿 | 勉强通过 | 高 |
| 小红书文案 | 基本通过 | 中 |
| 封面图提示词 | 通过 | 低 |
| Quality Check | 修正后可用 | 中 |

## 最终判断

Brand OS 已经能压住大部分内容输出，不会轻易滑回普通自媒体模板。

但要记住：

> Brand OS 不是让内容没有传播力。  
> 它是让传播力服从治理文明。

# 9. 下一步建议

可以进入官网首页制作。

但建议首页制作时：

1. 不使用直播转化稿作为首页文案来源。
2. 首页文案优先参考 `brand_os/brand_worldview.md`、`brand_os/visual_philosophy.md`、`brand_os/brand_voice.md`。
3. 首页只使用短句判断，不使用完整内容 Agent 的长文输出。
4. CTA 使用“进入档案 / 阅读信托制物业 / 了解治理体系 / 开始一次项目诊断”，不要使用“立即报名 / 免费咨询 / 领取资料”。
5. 官网首页第一阶段只做 Hero、第一段空间、光影、字体和滚动节奏，先建立世界。
