# Brand OS｜信托制物业品牌操作系统

Brand OS 是 `content-factory` 的品牌前置层。

它不是新的内容 Agent，也不是新的功能模块。它的作用是：在任何内容生成之前，先统一世界观、语言气质、情绪节奏、视觉哲学和禁止规则，防止内容在长期生产中变成普通自媒体、课程销售页、SaaS 官网或行业培训话术。

## 核心判断

信托制物业不是一个物业服务产品。

它是一套关于“信任如何被重新建立”的新治理文明。

内容、官网、视频、书籍、课程、咨询和社群，都必须服务于同一个底层叙事：

> 信任不是被说服出来的。  
> 信任是被记录、公开、验证和持续兑现出来的。

## 使用方式

所有 Agent 在生成内容前，必须先遵守本目录规则。

默认运行时只注入短版 Brand OS，节省 token。

完整品牌训练可在需要重写核心资产、官网、片头、书籍介绍、品牌手册时读取全量文件。

## 文件说明

- `brand_worldview.md`：世界观与价值主张。
- `brand_voice.md`：语言气质与句式规则。
- `emotion_rhythm.md`：情绪推进和纪录片旁白节奏。
- `visual_philosophy.md`：视觉哲学、空间、光影、材质。
- `brand_personality.md`：品牌人格。
- `forbidden_rules.md`：绝对禁止事项。
- `brand_os_compact.md`：低 token 注入版 Brand OS。

## 最高原则

不要把它写成“卖课的”。

要让用户感觉：

> 他们不是在做一个项目。  
> 他们在重新定义物业行业的信任秩序。
