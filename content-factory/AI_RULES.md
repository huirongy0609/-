# AI 内容生产规则

这个文件定义 `content-factory` 项目的内容生产标准。所有 Agent 和后续自动化脚本都必须遵守。

## 1. 所有内容必须先生成结构，再生成正文

任何内容都不能直接进入散文式输出。必须先明确：

- 目标人群
- 核心痛点
- 内容角度
- 结构顺序
- 转化方向

然后再生成完整正文。

## 2. 公众号文章必须使用七段结构

公众号文章固定使用以下七段：

1. 封面图提示词预留
2. 开头钩子
3. 问题放大
4. 误区拆解
5. 方法框架
6. 案例化表达
7. 总结转化

不得只输出提纲，必须输出可直接发布的完整正文。

## 3. 公众号文章开头要预留封面图提示词

公众号正文最前面必须包含：

```markdown
> 封面图提示词：
> ...
```

提示词必须包含人物、场景、情绪、构图、颜色和文字留白。

## 4. 短视频口播稿必须有自然转化收口

短视频稿不能只讲观点，必须在结尾自然引导用户下一步行动，例如：

- 关注
- 评论
- 收藏
- 查看长文
- 进入直播
- 领取资料
- 了解课程或服务

转化必须顺着内容逻辑发生，不能突然硬插广告。

## 5. 直播稿必须有卖书、卖课或服务的自然转化

直播转化稿必须至少包含以下一种方向：

- 卖书
- 卖课
- 咨询
- 陪跑
- 社群
- 服务

如果用户没有指定产品，默认生成卖书、卖课、咨询或服务三个版本。

不得承诺保证赚钱、保证涨粉、保证转化、保证效果。

## 6. 所有输出必须方便复制使用

所有内容必须使用清晰 Markdown 结构：

- 标题明确
- 段落清楚
- 模块独立
- 可直接复制到平台
- 不夹杂无关解释
- 不输出“以下是为你生成的”等多余废话

## 7. 不要输出空泛理论，要输出可直接发布的内容

禁止只写：

- 提纲
- 原则
- 方法论口号
- 抽象建议
- 没有场景的观点

必须输出：

- 完整标题
- 完整正文
- 完整口播
- 完整直播话术
- 完整小红书文案
- 完整封面提示词
- 完整分镜表
- 完整发布清单

## 8. 内容风格标准

内容要适合中文自媒体、短视频、公众号和直播带货场景：

- 有冲突感
- 有情绪感
- 有传播感
- 有具体场景
- 有真实人群
- 有明确行动

但不能：

- 低俗
- 恐吓
- 造谣
- 过度标题党
- 虚假承诺
- 侵犯隐私
- 诱导不理性消费

## 9. 最终内容包必须包含的模块

最终 `outputs/content_package.md` 必须包含：

1. 爆款选题分析
2. 标题
3. 公众号长文
4. 短视频口播稿
5. 直播转化稿
6. 小红书文案
7. 封面图提示词
8. 视频分镜脚本
9. 最终发布清单

缺少任何一项，都不能视为完成。
