# SYSTEM FREEZE｜系统冻结说明

当前阶段：稳定生产阶段。

从现在开始，本项目停止继续扩展新模块。唯一目标是降低运行成本，提高稳定性，并持续把内容发布和数据记录跑起来。

## 1. 当前系统已有能力

### 内容生产

- 读取 `inputs/topic.txt` 或命令行 `--topic`。
- 调用 OpenAI API。
- 生成标题、公众号长文、短视频口播、小红书、直播稿、封面提示词、视频分镜。
- 支持 `--lite` 低成本模式，只生成标题、公众号长文、短视频口播。

### 风格控制

- `style_training/` 提供中文头部自媒体风格库。
- 默认 `LOW_TOKEN_MODE` 只注入短风格规则，避免每个 Agent 重复读取完整风格库。

### 数据复盘

- `analytics/` 支持本地评分、爆款结构识别、表现日志记录。
- 数据复盘只做一次，不触发自动重写。

### 运营系统

- `operations/` 包含发布日历、平台策略、账号矩阵、内容数据跟踪表、爆款案例库、用户画像。

### 商业化系统

- `monetization/` 包含 Offer、产品阶梯、转化漏斗、信任系统、引流产品、咨询脚本、社群策略、销售内容模板、90 天路线图。

## 2. 当前禁止继续扩展的模块

禁止新增以下类型模块：

- 新 Agent 模块。
- 新 prompt 体系。
- 新自动重写引擎。
- 新评分循环。
- 新递归优化器。
- 新运营文档层。
- 新商业化文档层。
- 大型重构。
- 自动发布模块。
- 自动抓热点模块。
- 自动私域互动模块。

这些模块可以在未来单独立项，但稳定生产阶段不再加入。

## 3. 当前唯一目标

### 稳定生成

- 每个 Agent 最多执行 1 次。
- quality_check 最多执行 1 次。
- 默认启用 LOW_TOKEN_MODE。
- 默认不做自动重写。
- 默认不做无限评分和优化。

### 稳定发布

- 按 `operations/publish_calendar.md` 执行。
- 用 `--lite` 生成日常内容。
- 只在需要完整内容包时运行标准模式。

### 稳定收集数据

- 发布后人工或脚本填写 `operations/content_tracking.csv`。
- 高表现内容写入 `operations/viral_case_library.md`。
- 每周复盘一次，不做实时无限优化。

## 4. 推荐运行方式

日常生产：

```text
.venv/bin/python run_pipeline.py --lite
```

完整内容包：

```text
.venv/bin/python run_pipeline.py
```

指定主题：

```text
.venv/bin/python run_pipeline.py --lite --topic "普通人如何用 AI 提升内容生产效率"
```

## 5. 成本控制原则

- 不重复读取整个项目。
- 不把所有上游内容传给所有 Agent。
- 不把完整风格库注入每次调用。
- 不做失败后自动重写。
- 不在一次运行里追求完美，只追求可发布草稿。
- 质量提升放到人工复盘和下一次选题中完成。
