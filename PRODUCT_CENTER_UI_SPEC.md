# PRODUCT_CENTER_UI_SPEC

本文件定义 `产品中心` UI 规范。

本阶段仅输出设计规范，不涉及开发。

## 1. 产品中心定位

产品中心是平台产品统一入口。

它负责：

```text
产品展示
产品销售
产品服务
商业化转化
```

产品中心不是：

```text
淘宝店
课程商城
知识店铺
软件官网
促销页
```

## 2. 设计目标

产品中心应让用户理解：

- 平台有哪些产品与服务。
- 每类产品解决什么问题。
- 适合哪些用户。
- 如何进一步咨询、申请或报名。

重点是产品价值，不是商品陈列。

## 3. 视觉参考

参考：

- Stripe：产品矩阵和解决方案表达。
- Apple：产品价值和克制视觉。
- Linear：专业工具表达。
- Notion：模板与知识资源组织。

禁止：

- 淘宝风。
- 课程商城风。
- 知识店铺风。
- 价格表堆砌。
- 促销倒计时。

## 4. 页面结构

建议结构：

```text
顶部：产品中心定位
产品矩阵：图书 / 训练营 / 专题课程 / 软件 / 工具 / 未来产品
重点产品：当前最重要产品
解决方案：按用户问题组织
适用人群：物业 / 社区 / 业委会 / 街道 / 研究者
咨询与申请：轻量入口
```

## 5. 产品分类 UI

分类：

```text
图书产品
训练营产品
专题课程
软件产品
工具产品
未来产品
```

分类卡片应包含：

```text
产品类型
解决的问题
适合对象
代表产品
下一步动作
```

## 6. 图书产品 UI

当前规划：

```text
《信托制物业》
《信托制社区治理》
《金牌管家成长手册》
未来图书
```

UI重点：

- 书名。
- 核心主题。
- 适合读者。
- 与内容中心/成长世界的关系。

禁止：

- 电商商品详情风。
- 大促销按钮。
- 价格优先。

## 7. 训练营产品 UI

当前规划：

```text
成都研学营
项目经理训练营
社区书记训练营
物业高管训练营
未来训练营
```

UI重点：

- 训练目标。
- 适合对象。
- 训练内容。
- 交付形式。
- 申请入口。

训练营应像专业项目，不像旅游产品或课程商品。

## 8. 专题课程 UI

当前规划：

```text
公共收益专题课
预算管理专题课
业委会专题课
未来课程
```

UI重点：

- 解决的问题。
- 学完能做什么。
- 适合谁。
- 与成长路径的关系。

禁止：

- 课程封面墙。
- 爆款课文案。
- 知识付费店铺风。

## 9. 软件产品 UI

当前规划：

```text
信托制物业管理软件
未来数字治理工具
```

UI重点：

- 软件解决的治理问题。
- 支持的信托制场景。
- 与制度公开、预算管理、公共收益、协同治理的关系。
- 申请演示或咨询入口。

禁止：

- SaaS后台截图堆砌。
- 企业官网式夸张功能清单。

## 10. 工具产品 UI

当前规划：

```text
制度模板包
治理工具包
表单工具包
资料包
```

UI重点：

- 使用场景。
- 适用角色。
- 包含内容。
- 获取方式。

工具包应像可信资料包，不像下载站。

## 11. 未来产品 UI

当前规划：

```text
认证产品
顾问服务
项目诊断服务
城市辅导服务
```

UI重点：

- 服务对象。
- 服务价值。
- 适用场景。
- 申请沟通入口。

## 12. 产品详情页原则

产品详情页应回答：

```text
这个产品解决什么问题
适合谁
包含什么
如何使用
如何申请
相关内容和制度资源是什么
```

不应只回答：

```text
多少钱
怎么买
有什么优惠
```

## 13. 与其他模块关系

产品中心连接：

- 内容中心：图书、课程、工具的内容基础。
- 制度中心：模板包、工具包来源。
- 成长世界：训练营、课程、认证的成长路径关系。
- 用户识别系统：根据画像推荐产品。
- AI信托助手：帮助找产品、找课程、生成材料。

## 14. 产品中心视觉禁区

禁止：

- 淘宝风。
- 店铺装修风。
- 课程超市。
- 价格卡片墙。
- 促销标语。
- 大红大黄营销色。
- 过度销售话术。
