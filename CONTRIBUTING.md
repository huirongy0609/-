# Contributing

Status: Draft
Version: v0.1.0
Date: 2026-07-12

## 基本原则

本仓库以 GitHub 作为统一项目办公室。

任何正式成果必须进入仓库、Issue、Pull Request、Project 或交接文件，不再只保留在聊天记录中。

## 工作入口

1. 新事项先建任务单或 Issue。
2. 任务必须明确 Owner、Reviewer、Approver。
3. 内容生产交给 Work。
4. 工程、自动化、部署和数据结构交给 Codex。
5. ChatGPT 负责拆解、协调、审核和一致性检查。
6. 重大方向、标准和争议由杨慧荣批准。

## 分支规则

- 不直接修改主分支。
- 每个非 trivial 任务使用独立分支。
- 分支名应能看出任务目的。
- 涉及已批准文件正文的修改必须通过 Pull Request。

## 文件状态

正式文件至少使用以下状态：

- Draft
- Review
- Approved
- Superseded
- Archived

已批准文件不得由 Codex 直接修改正文。如需调整，必须新建分支、提交变更说明、形成 PR、标明影响范围并等待审核批准。

## Pull Request 要求

PR 必须说明：

- 本次修改目的。
- 修改文件清单。
- 是否影响已批准内容。
- 是否影响网站。
- 是否影响数据结构。
- 是否需要内容审核。
- 是否需要项目总架构师批准。
- 测试结果。
- 回滚方案。

## 禁止事项

- 不得擅自重写专业定义。
- 不得把 mock 数据伪装为真实数据。
- 不得删除、迁移或覆盖现有文件，除非任务明确批准。
- 不得绕过任务、交接和决策记录修改平台底层结构。
