# PRODUCT_CENTER_STRUCTURE

本文件定义 `产品中心` 的结构。本阶段仅做规划，不进行开发。

## 1. 定位

产品中心是 `信托制社区治理平台` 的产品统一入口。

它负责：

- 产品展示
- 产品销售
- 产品服务
- 商业化转化

产品中心不是成长世界。

成长世界负责育人，产品中心负责产品与服务转化。

## 2. 与成长世界的分工

```text
金牌管家成长世界
  → 人才成长
  → 学习训练
  → 成长任务
  → 成长档案
  → AI陪练
  → AI考官
  → 职业成长路径

产品中心
  → 产品展示
  → 产品销售
  → 产品服务
  → 商业化转化
```

边界原则：

- 成长路径、训练任务、成长档案、能力评价归入成长世界。
- 图书、训练营报名、专题课程售卖、软件、工具包、顾问服务归入产品中心。
- 同一内容可以同时服务成长和产品，但入口职责必须分清。

## 3. 产品中心一级结构

```text
图书产品
训练营产品
专题课程
软件产品
工具产品
未来产品
```

## 4. 图书产品

当前规划：

```text
《信托制物业》
《信托制社区治理》
《金牌管家成长手册》
未来图书
```

用途：

- 传播信托理念。
- 沉淀治理经验。
- 建立行业认知入口。
- 为内容中心和成长世界提供基础读物。

## 5. 训练营产品

当前规划：

```text
成都研学营
项目经理训练营
社区书记训练营
物业高管训练营
未来训练营
```

用途：

- 承接线下学习与参访。
- 服务项目经理、社区书记、物业高管等关键人群。
- 将平台内容、制度和案例转化为可交付训练产品。

## 6. 专题课程

当前规划：

```text
公共收益专题课
预算管理专题课
业委会专题课
未来课程
```

用途：

- 针对具体治理问题提供短周期学习产品。
- 与制度中心、治理实验室形成关联。
- 为成长世界提供可复用学习内容。

## 7. 软件产品

当前规划：

```text
信托制物业管理软件
未来数字治理工具
```

用途：

- 支撑信托制物业落地。
- 服务制度公开、预算管理、公共收益、协同治理等场景。
- 作为后续商业化产品线。

## 8. 工具产品

当前规划：

```text
制度模板包
治理工具包
表单工具包
资料包
```

用途：

- 帮助用户快速获得可落地材料。
- 与制度中心形成强关联。
- 支持物业、社区、业委会、街道等不同角色使用。

## 9. 未来产品

当前规划：

```text
认证产品
顾问服务
项目诊断服务
城市辅导服务
```

用途：

- 承接更高价值服务。
- 服务机构、城市、项目和人才。
- 与用户识别与资源连接系统形成转化闭环。

## 10. 首页呈现原则

首页需要展示产品中心入口，但不能变成产品商城。

首页应体现：

- 平台内容
- 平台制度
- 成长世界
- 平台产品
- 用户需求入口

首页禁止：

- 产品商城化
- 课程商城化
- 软件官网化
- 价格表堆砌
- 促销式文案

## 11. 当前项目映射

| 当前资产 | 映射方式 |
|---|---|
| 首页“成长认证中心”相关表达 | 拆分：成长路径归成长世界，产品转化归产品中心 |
| 停车治理 Demo | 可作为专题课程、训练营或工具包的内容素材 |
| 案例库 | 可作为图书、课程、训练营和工具包素材 |
| 制度规则提示 | 可转化为制度模板包、治理工具包 |
| 提交页 | 可承接训练营报名、学习申请、项目诊断申请等需求 |

## 12. 阶段边界

本阶段只规划产品中心结构。

禁止：

- 开发产品中心页面
- 新增产品路由
- 新增购买流程
- 新增支付
- 新增订单系统
- 新增产品数据库
- 把首页改成商城
