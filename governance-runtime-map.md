# Governance Runtime Map

运行层：城市运行态势层

目标：构建全国城市治理神经网络，让用户感觉城市正在被 AI 协同感知。

## 1. 核心定位

城市运行态势层不是地图页。

它是：

**全国城市治理信号的实时感知空间。**

## 2. 核心对象

- 城市节点
- 社区热区
- 风险信号
- 舆情波动
- 政策解析
- 案例同步
- 共建机构
- AI 扫描路径

## 3. 运行态表现

必须包含：

- 城市节点缓慢呼吸。
- 风险波纹低透明扩散。
- 光点沿治理网络迁移。
- AI 扫描缓慢推进。
- 城市群协同脉冲。
- 实时治理信号进入队列。

## 4. 信息结构

推荐结构：

```text
左侧：运行态摘要 / 在线城市 / 风险事件 / AI 研判队列
中央：全国城市治理神经网络
右侧：实时治理信号 / 风险关注 / 城市节点详情
底部：城市协同指标 / 数据迁移状态
```

## 5. 视觉规则

- 背景必须是深空层。
- 地图必须像数字孪生空间，不像静态地图。
- 节点颜色以 Civic Teal 为主。
- 风险以低饱和 Amber 表达。
- 红色只用于极少量高风险。
- 不使用蓝色科技大屏。

## 6. Mock Realtime Data

第一版可使用模拟数据：

- AI 正在扫描常州、成都、杭州、深圳等城市。
- 公共收益透明度事件进入风险研判。
- 老旧小区停车治理案例正在同步。
- 政策动态正在进入解析队列。
- 城市共建节点在线变化。

## 7. 验收标准

用户打开该运行层时，第一感觉必须是：

**全国城市治理网络正在运行。**

如果看起来只是地图页或后台页面，失败。
