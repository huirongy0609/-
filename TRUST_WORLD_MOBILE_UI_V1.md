# TRUST_WORLD_MOBILE_UI_V1

本文件定义 `Trust World（信托制世界）` 第一版手机端产品原型。

本阶段目标不是最终 UI，不是视觉稿，不是开发实现，而是验证：

```text
用户第一次进入信托制世界是否顺畅
逻辑是否成立
路径是否成立
```

设计尺寸：

```text
390 x 844
```

## 1. 产品定位

Trust World 不是：

```text
新闻网站
政务网站
研究院官网
资料下载站
```

Trust World 是：

```text
中国信托制实践入口
```

用户进入后应该感觉：

```text
我正在进入一个信托制世界
```

而不是：

```text
我打开了一个机构官网
```

## 2. 手机端总路径

```text
首次访问
  ↓
第一屏：欢迎来到信托制世界
  ↓
第二屏：中国信托制实践地图
  ↓
第三屏：选择我的身份
  ↓
第四屏：我的信托制世界
  ↓
第五屏：Resource Hub
  ↓
第六屏：Growth World 导流
```

## 4. 全局手机端框架

### 4.1 顶部

```text
左侧：Trust World
副标：信托制世界
右侧：身份入口 / 菜单
```

### 4.2 底部悬浮导航

V1 建议保留 4 个底部入口：

```text
首页
地图
身份
资源
```

说明：

- `首页` 回到第一屏。
- `地图` 进入中国信托制实践地图。
- `身份` 进入角色选择。
- `资源` 进入 Resource Hub。

Growth World 不放在底部固定导航里，而是在合适角色和合适场景中出现导流。

## 16. Visual System A

Trust World 视觉关键词：

```text
公开
透明
可信
专业
```

参考：

```text
Apple
Notion
Airbnb
McKinsey
```

品牌色：

```text
高级灰调青绿色
```

视觉原则：

- 大量留白。
- 内容清晰。
- 卡片克制。
- 地图高级但不科技炫光。
- 角色入口温和直接。
- 不像机构官网。

禁止：

```text
新闻网站风
政务网站风
研究院官网风
资料下载站风
驾驶舱
科技蓝
发光边框
复杂动效
```

## 17. V1 验证问题

本原型用于验证以下问题：

```text
用户第一次进入是否知道这是信托制世界？
用户是否理解为什么会有信托制？
用户是否能看到全国实践正在发生？
用户是否能快速找到自己的身份？
用户是否能进入属于自己的信托制世界？
用户是否知道下一步是资源连接还是成长世界？
```

如果以上问题不能被顺畅回答，说明 Trust World Mobile V1 还不成立。
