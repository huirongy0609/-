# PLATFORM_UI_SYSTEM

本文件定义 `信托制社区治理平台` 的整体 UI 设计系统。

本阶段仅输出设计规范，不涉及开发。

## 1. 设计定位

平台气质：

```text
专业
可信
温暖
内容感
成长感
长期主义
```

平台不是：

```text
政府网站
物业后台
数据驾驶舱
SaaS管理系统
ERP系统
传统CMS
知识付费商城
```

平台应该像：

```text
高质量行业平台
内容平台
成长平台
资源连接平台
信托制社区治理知识与人才基础设施
```

## 2. 视觉参考

重点参考：

- Apple：留白、秩序、克制、产品表达。
- Notion：内容组织、轻量结构、知识感。
- Linear：清晰层级、专业感、极简交互。
- Stripe：产品矩阵、服务表达、商业可信感。
- Arc Browser：温暖科技感、柔和界面气质。
- 得到：知识产品、学习路径、内容权威感。

禁止参考：

- 数据驾驶舱
- 大屏系统
- 政府门户
- 物业后台
- 传统CMS
- 知识付费商城

## 3. 整体设计语言

关键词：

```text
留白
秩序
温度
可信
长期主义
内容优先
成长路径
资源连接
```

页面应有“行业平台”的稳定感，而不是“运营后台”的控制感。

信息应像被编辑过的内容，而不是从数据库直接倒出来的字段。

界面要让用户感觉：

- 这里有长期沉淀的行业知识。
- 这里有可信的制度和工具。
- 这里能帮助自己成长。
- 这里能连接到真实资源。

## 4. 色彩系统

### 主色

灰调青绿色。

用途：

- 品牌识别
- 主按钮
- 重点链接
- 选中状态
- 成长路径节点
- 资源连接提示

建议方向：

```text
深青绿
灰青绿
柔和薄荷绿
低饱和松石绿
```

禁止：

```text
高饱和科技蓝
荧光绿
赛博紫
大面积渐变
强发光色
```

### 中性色

用于正文、背景、边框和卡片层级。

建议方向：

```text
暖白
浅灰
石墨灰
雾灰
温润黑
```

背景不应纯白刺眼，也不应全黑科技化。

### 辅助色

用于内容类型、状态提示和轻微区分。

建议方向：

```text
制度：灰青
内容：墨灰
成长：温和绿
产品：暖米灰
资源连接：低饱和金绿
提醒：温和琥珀
风险：克制砖红
```

注意：

辅助色只做轻提示，不做大色块。

## 5. 字体系统

### 字体气质

字体应体现：

- 专业
- 清晰
- 可阅读
- 不花哨

中文优先使用系统默认无衬线。

### 字级层级

建议层级：

```text
H1：平台主标题
H2：页面主标题
H3：区块标题
H4：卡片标题
Body：正文
Meta：栏目、来源、日期、状态
Caption：说明、注释、辅助信息
```

### 排版原则

- 标题不应过度巨大。
- 正文阅读宽度应克制。
- 内容页面应优先保证可读性。
- 不用负字距。
- 不用过度装饰字体。
- 中文段落行高应舒适。

## 6. 间距系统

页面应有明显留白。

建议节奏：

```text
页面外边距：宽松
区块间距：明显
卡片间距：中等
段落间距：舒适
表单项间距：清晰
```

原则：

- 首页靠留白建立可信感。
- 内容中心靠阅读宽度建立沉浸感。
- 成长世界靠路径间距建立节奏感。
- 产品中心靠矩阵间距建立秩序感。
- 用户识别流程靠单步聚焦减少压力。

## 7. 布局系统

### 页面宽度

建议采用三类容器：

```text
窄容器：文章阅读、制度说明、表单说明
中容器：首页区块、内容列表、成长路径
宽容器：产品矩阵、资源连接、治理实验室
```

### 栅格

优先使用简单栅格：

```text
1列：文章、表单、用户识别流程
2列：首页重点区块、产品介绍
3列：栏目卡片、产品矩阵
4列：轻量入口卡片
```

禁止：

- 高密度后台布局
- 多列小字堆砌
- 图表墙
- 统计卡片墙

## 8. 组件系统

### 导航

导航应清晰、克制、内容平台化。

一级导航：

```text
首页
内容中心
制度中心
成长世界
产品中心
资源连接
治理实验室
```

AI信托助手不作为一级导航，可作为全站搜索/助手入口常驻。

### 卡片

卡片应像“内容摘要”或“产品说明”，不应像后台面板。

卡片类型：

```text
内容卡片
制度卡片
成长任务卡片
产品卡片
资源连接卡片
案例样板卡片
```

卡片原则：

- 少边框。
- 轻阴影或无阴影。
- 用标题、摘要、标签、来源构成信息。
- 不用发光边框。
- 不用数字大屏卡片。

### 按钮

按钮用于明确行动。

按钮类型：

```text
主按钮：进入、申请、开始画像
次按钮：查看、了解、收藏
文本按钮：阅读更多、查看详情
辅助按钮：筛选、排序、切换栏目
```

按钮文案应具体：

```text
开始建立画像
进入成长世界
查看制度模板
申请参访
提交项目需求
```

避免：

```text
立即体验
马上领取
限时抢购
进入后台
查看数据
```

### 标签

标签用于内容组织。

标签类型：

```text
栏目标签
主题标签
角色标签
城市标签
阶段标签
需求标签
```

标签视觉应轻，不应抢正文。

### 表单

表单应像“对话式识别”，而不是后台录入。

原则：

- 一次只问必要信息。
- 字段语言自然。
- 说明用户为什么填写。
- 让用户感到被理解，而不是被审核。

## 9. 页面原则

### 首页

首页是行业平台门户。

必须回答：

- 最近发生了什么。
- 有哪些值得关注的实践。
- 有哪些制度资源。
- 如何进入成长世界。
- 如何连接资源。

### 内容中心

内容中心是阅读和沉淀空间。

必须重视：

- 标题质量
- 摘要质量
- 来源与时间
- 栏目关系
- 阅读连续性

### 制度中心

制度中心是可复用工具空间。

必须重视：

- 文件类型
- 适用对象
- 使用场景
- 下载/复制/申请路径

### 成长世界

成长世界是成长操作系统。

必须重视：

- 阶段感
- 任务感
- 进度感
- 成就感
- 陪伴感

### 产品中心

产品中心是产品与服务入口。

必须重视：

- 产品矩阵
- 适用人群
- 解决问题
- 产品价值
- 后续咨询/申请路径

禁止商城化和促销化。

### 用户识别与资源连接系统

必须重视：

- 首次进入体验
- 用户画像建立
- 推荐逻辑解释
- 需求提交
- 资源连接反馈

## 10. 交互原则

交互应轻。

允许：

- 轻微 hover
- 平滑展开
- 栏目切换
- 标签筛选
- 分步识别
- 保存阅读状态

禁止：

- 炫技动画
- 大屏动效
- 高频闪烁
- 复杂后台操作
- 拖拽式管理系统

## 11. 文案原则

文案应专业、可信、温暖。

优先使用：

```text
了解
学习
沉淀
连接
申请
参与
成长
共建
公开透明
价值共享
```

避免使用：

```text
管理
监控
驾驶舱
指挥
运营后台
立刻购买
限时优惠
爆款课程
```

## 12. 禁止事项

全站禁止：

- 驾驶舱风格
- 后台风格
- 大屏风格
- 蓝色科技风
- 发光边框
- 炫光效果
- 密集统计卡片
- 产品商城风
- 课程超市风
- 传统CMS风
