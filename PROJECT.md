# Project Governance

## Product Mission

《全国社区治理协同地图》是社区治理协同平台的第一版入口。

它用于展示全国社区治理、信托制物业、老旧小区治理、AI 行业情报和案例共建网络，让政府、街道、物业、业委会、社工机构、专家和合作方看到一个可持续扩展的全国协同平台雏形。

第一版不是正式业务系统，而是行业宣言和平台入口。

## Target Users

### Primary Users

- 街道/社区治理相关人员
- 社工机构
- 物业公司
- 业委会和居民自治组织
- 社区治理专家顾问
- 研究机构
- 政府和合作方演示对象

### Secondary Users

- 行业服务商
- 城市共建伙伴
- 政策和行业研究人员
- 平台运营人员

## Platform Boundary

This project is:

- 全国社区治理协同入口
- 案例沉淀平台
- AI 行业情报展示平台
- 城市共建网络入口
- 信托制物业和透明治理的行业表达载体

This project is not:

- 物业管理系统
- SaaS 后台
- 社区 App
- 工单系统
- 缴费系统
- 课程系统
- 考试认证系统
- CRM
- 爬虫平台
- 复杂 AI Agent 平台

## Current MVP Scope

Allowed:

- 首页
- 全国地图/城市协同示意
- AI 情报信息流
- 案例库与案例详情
- 共建提交表单演示
- 本地 mock JSON 数据
- 克制、专业、演示友好的 UI

Not allowed without architecture review:

- 登录
- 支付
- 复杂权限
- 复杂后台
- 真实 AI 接口
- 真实爬虫
- 真实上传
- 数据库写入
- 重型状态管理

## Directional Guardrails

- 所有新功能必须服务“全国协同、案例沉淀、AI 情报、开放共建”。
- 不要把产品带向物业日常管理。
- 不要把产品带向课程/认证/社群售卖。
- 不要用高饱和科技风替代政务克制感。
- 不要为了演示效果引入不可维护的复杂动画。
- 不要把 mock 数据伪装成真实数据。

## Product Principle

第一版不是正式产品。

第一版要让人第一眼感受到：

- 全国正在发生社区治理变革。
- 信托制物业和透明治理正在扩散。
- 平台可以成为全国协同入口。
- AI 可以持续整理行业情报。
- 全国参与者可以共同提交案例和动态。

## Architecture Principle

先建立清晰领域模型，再扩展页面。

先建立设计系统，再扩展视觉。

先治理仓库边界，再增加依赖。

先保留 MVP 简洁性，再引入工程复杂度。
