# USER_SYSTEM_UI_SPEC

本文件定义 `用户识别与资源连接系统` UI 规范。

本阶段仅输出设计规范，不涉及开发。

## 1. 系统定位

用户识别与资源连接系统不是普通用户中心。

它的核心逻辑是：

```text
先识别用户
再提供内容
再推荐成长
再推荐产品与资源
最后完成连接
```

## 2. 参考产品

参考：

- Keep：先识别状态，再推荐训练。
- B站：先识别兴趣，再推荐内容。
- LinkedIn：先识别身份与职业关系，再连接资源。

不参考：

- 后台用户中心。
- 会员中心。
- CRM表单。
- 政务注册系统。

## 3. 首次进入体验

首次进入不应直接要求完整注册。

建议流程：

```text
欢迎说明
选择身份
选择城市
填写单位
选择当前状态
选择关注方向
选择需求方向
生成推荐入口
```

体验原则：

- 轻量。
- 分步。
- 解释用途。
- 允许跳过部分问题。
- 不制造审核压力。

## 5. 用户画像建立流程 UI

建议分步：

### Step 1：你是谁

选择身份。

UI形式：

- 身份卡片。
- 每个身份有一句说明。
- 不使用下拉框优先。

### Step 2：你在哪里

选择城市，填写单位。

UI形式：

- 城市输入或选择。
- 单位可选填。
- 说明用于推荐本地内容和资源。

### Step 3：你当前处在什么状态

选择当前状态。

UI形式：

- 多选或单选卡片。
- 文案贴近真实问题。

### Step 4：你关注什么

选择关注方向。

UI形式：

- 标签组。
- 可多选。
- 控制数量，避免过密。

### Step 5：你现在需要什么

选择需求方向。

UI形式：

- 需求卡片。
- 明确下一步会推荐什么。

### Step 6：生成推荐

输出：

```text
推荐内容
推荐制度
推荐成长任务
推荐产品
推荐资源连接方式
```

## 6. 推荐结果 UI

推荐结果不应像算法信息流。

应像“为你整理好的入口”。

推荐区块：

```text
为你推荐的内容
适合你的制度工具
建议进入的成长任务
可能需要的产品与服务
可连接的资源
```

每条推荐应说明：

```text
为什么推荐
适合谁
下一步做什么
```

## 7. 需求提交流程 UI

需求提交不是后台表单。

需求类型：

```text
案例投稿
项目需求
参访申请
讲课邀请
学习申请
产品咨询
资源连接
```

流程：

```text
选择需求类型
补充必要信息
确认联系方式
平台反馈下一步
```

表单原则：

- 字段少。
- 说明清楚。
- 不一次性要求过多证明材料。
- 不像政务审批。

## 8. 资源连接流程 UI

资源连接应体现平台价值。

连接对象：

```text
内容资源
制度资源
课程训练
产品服务
专家顾问
机构合作
城市实践
项目机会
```

UI形式：

- 根据画像推荐。
- 每个资源说明适配原因。
- 提供“申请连接”或“了解更多”入口。

## 9. 用户主页 UI

用户主页不是会员中心。

建议展示：

```text
我的画像
我的推荐
我的成长任务
我的收藏内容
我的申请
我的资源连接
我的成长档案入口
```

不展示：

- 积分商城。
- 充值会员。
- 复杂权限。
- 后台菜单。

## 10. 情感与语气

系统语气应像：

```text
我先了解你是谁，再帮你找到合适的内容、成长路径和资源。
```

不应像：

```text
请完善资料，否则无法使用系统。
```

## 11. 与其他模块关系

用户识别系统连接：

- 内容中心：推荐内容。
- 制度中心：推荐制度和模板。
- 成长世界：推荐成长任务。
- 产品中心：推荐产品与服务。
- 治理实验室：推荐样板案例。
- AI信托助手：根据画像帮用户找材料和生成材料。

## 12. 视觉禁区

禁止：

- CRM风格表单。
- 政务审批流程。
- 会员中心风。
- 复杂账号后台。
- 大量必填字段。
- 强制注册压迫感。
