# 《信托制物业导读》系列视频制作规范

## 项目定位

本项目用于制作“信托制物业导读”系列内容。内容以《信托制物业》全书导读为主线，面向物业公司、项目经理、业主委员会、物管会、社区工作者、基层治理部门、社区治理推动者和物业行业创业者。

视频不是普通读书笔记，也不是硬广宣传片，而是用一套高级、清爽、动态化的知识视频语言，讲清信托制物业的制度逻辑、落地路径和金牌管家体系。

## 默认视觉风格

采用 Gamma/Canva 的清爽高级排版，加上 HyperFrames 的动态镜头感。

整体方向：

- 浅色背景
- 青绿色玻璃质感
- 3D 信息块
- 卡片推进
- 关系图动态展开
- 法条浮层
- 对照表切换
- 局部放大
- 进度感转场
- 清晰、现代、可信赖的社区治理知识片风格

明确避免：

- 普通静态 PPT 翻页
- 黑底科技风
- 低质感模板风
- 纯字幕堆砌
- 只有 HTML 页面或普通幻灯片截图的假视频

## 默认视频结构

默认采用：

动态 PPT 主画面 + 右下角圆形真人口播形象。

主画面负责呈现：

- 概念解释
- 篇章结构
- 法条依据
- 权责关系
- 资金流向
- 制度对照
- 表单文件
- 软件界面
- 金牌管家标准化体系

右下角圆形真人口播窗口负责辅助讲解，不抢主画面。

人物窗口原则：

- 位置：右下角
- 形态：圆形或柔和圆角浮窗
- 占比：辅助位置，不超过主画面的 15%-20%
- 不遮挡主标题、关键图表、法条和数据

## 生产逻辑

以后不按录音倒推 PPT，而是采用文字驱动流程。

标准流程：

原文/主题 → 连续口播稿 → 克隆声音 → 数字人/头像口播 → 动态 PPT 视频 → 字幕合成 → 成片

也就是说，每一条视频先完成可直接录制的连续口播稿，再由口播稿拆分动态画面，而不是先做 PPT 再补声音。

## 克隆音色规范

默认使用讲解者本人声线进行克隆配音。

声音风格：

- 自然
- 清楚
- 沉稳
- 有讲课感
- 语速中等偏稳
- 不过度播音腔
- 不过度情绪化
- 不像广告配音

内容属于物业治理、业主共同体、信托制物业、《民法典》《信托法》等知识讲解，配音应像一位专业老师在给业主、业委会、物业从业者做导读。

如果有克隆音色可用，优先使用克隆音色生成整条口播音频。如果暂时没有可用克隆音色，可以先生成标准普通话 TTS 试稿，但最终成片应优先替换为本人克隆音色。

现阶段可暂用已有声线素材作为参考。后续应尽量使用更干净的 5 到 8 分钟专门录音作为声线克隆素材。

## 克隆形象规范

默认使用讲解者本人形象作为视频中的辅助口播窗口。

人物不需要占满画面，不做全屏数字人讲课。推荐放在右下角、右侧或局部圆形窗口中，作为陪伴式讲解者出现。

主画面仍然以动态 PPT、关系图、法条卡片、资金流向图、制度对照表为核心。人物形象只承担“有人在讲”的信任感和亲切感，不抢主画面。

人物窗口建议：

- 圆形或柔和边界
- 尺寸稳定
- 位置固定或轻微动态变化
- 不频繁跳动
- 不遮挡标题、法条、关键结构图

如果数字人口型偶尔不完美，应通过缩小人物窗口、增强主画面信息密度、保留字幕等方式降低违和感。

## 字幕规范

字幕应跟随口播内容自动生成，默认使用中文简体。

字幕不要一整段铺满屏幕，应按语义短句分段显示。

字幕原则：

- 每屏字幕尽量控制在 1 到 2 行。
- 字幕服务理解，不抢主画面。
- 不遮挡法条、关系图、资金流向图和制度对照表。
- 关键金句可单独做成屏幕大字，普通字幕则保持轻量。

## 内容风格

语言要求：

- 通俗
- 准确
- 有现实问题感
- 有制度解释力
- 有法律依据但不堆砌法条
- 适合短视频和课程导读传播

面向读者：

- 物业公司
- 项目经理
- 业主委员会
- 物管会
- 街道社区
- 基层治理部门
- 社区治理顾问
- 物业行业创业者

涉及信托制物业时，必须注意：

- 区分民事信托逻辑和金融信托逻辑。
- 不把信托制物业讲成金融产品。
- 必要时引用《信托法》《民法典》相关法条。
- 法条用于支撑制度逻辑，不做机械堆砌。
- 重点讲清委托、受托、受益、财产独立、受托责任、公共事务管理等治理关系。

## 金牌管家体系露出原则

金牌管家体系不做硬广植入，而是通过工具和场景自然出现。

默认露出方式：

- 金牌管家信托制导入评估表
- 金牌管家小区治理问题诊断表
- 金牌管家业主共同基金预算表
- 金牌管家项目运营月报
- 金牌管家风险台账
- 金牌管家软件后台
- 金牌管家标准化文件夹体系
- 金牌管家人才培训路径
- 金牌管家项目复制模型

口播中可以自然出现：

- “在金牌管家的项目实践里，这一步通常不会靠经验判断，而是先做导入评估。”
- “成熟的信托制，不能只停留在理念，它必须落到表单、流程、系统和责任清单里。”
- “金牌管家体系做的，就是把这些看不见的治理逻辑，变成看得见、用得上的工具。”

避免出现：

- “赶紧选择金牌管家”
- “金牌管家最好”
- “马上加盟”
- “唯一方案”
- 过度广告化的推销语言

## 默认交付

如果用户要求“生成视频”，默认输出一条适合短视频平台传播的成片方案。

如条件允许，直接生成 MP4。

默认规格：

- 横屏：16:9，适合视频号、B站、抖音横屏、课程平台。
- 竖屏：9:16，适合抖音、视频号竖屏、小红书。
- 未特别说明时，优先按用户当前要求执行；本项目目前默认优先横屏 16:9。

成片必须具备：

- 动态 PPT 感
- 主画面结构化表达
- 右下角圆形口播人物
- 清晰字幕
- 章节进度感
- 金牌管家体系软露出

不能只交付：

- HTML 页面
- 普通 PPT
- 静态图片合集
- 无动态镜头的视频

## 质量判断标准

一条合格的视频应满足：

- 听起来像讲解者本人在讲，而不是陌生 AI 配音。
- 看起来像有设计感的动态课程短片，而不是普通 PPT 录屏。
- 人物形象增强信任感，但不干扰主画面理解。
- 画面节奏跟口播语义一致，不需要人工逐页对齐。
- 整套流程可以复用到下一篇内容，而不是每篇从零开始手工制作。

## 单条视频默认结构

1. 片头：品牌与标题，5-8 秒。
2. 开场问题：用一个现实痛点切入，10-20 秒。
3. 篇章定位：说明本篇在全书中的位置，15-25 秒。
4. 核心解释：用图表、关系图、对照表讲清本篇逻辑，2-4 分钟。
5. 金牌管家软露出：通过表单、系统或文件体系出现，20-40 秒。
6. 总结金句：提炼本篇最重要判断，10-15 秒。
7. 下一篇钩子：引导进入下一节，10 秒。
