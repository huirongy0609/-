# Project Office

Project Office 是中国信托制物业发展平台唯一正式协作中心。

所有任务、成果、交接均通过 GitHub 留痕，不再依赖聊天窗口传递正式文件。

## 协作流程

```text
ChatGPT
→ GitHub Inbox
→ Work / Codex
→ Submission
→ ChatGPT Review
→ Handoff
→ Archive
```

## 目录说明

```text
project-office/
├── inbox/          接收 ChatGPT 发布给 Work 或 Codex 的任务
├── submissions/    Work 或 Codex 提交待审核成果
├── handoffs/       任务完成后的正式交接记录
├── archive/        已归档任务、成果和交接
└── standards/      协作规范、模板和项目办公室标准
```

## 基本规则

- `inbox/work/`：提交给 Work 的任务。
- `inbox/codex/`：提交给 Codex 的任务。
- `submissions/work/`：Work 提交的待审核成果。
- `submissions/codex/`：Codex 提交的待审核成果。
- `handoffs/work/`：Work 的正式交接记录。
- `handoffs/codex/`：Codex 的正式交接记录。
- `archive/`：已完成或废止事项归档。
- `standards/`：项目办公室自身协作规则与模板。

除项目组明确要求外，本目录不存放业务正文、平台代码或未经审核的正式知识内容。
