# Real Dashboard Layouts

项目：社区治理协同平台

定位：城市级社区治理操作系统的真实可演示大屏布局。

目标：让首页、数据驾驶舱、地图演示页看起来像一个已经在运行的治理中枢，而不是装饰型科技大屏。

## 1. 总体布局原则

- 地图是主视野，不是背景装饰。
- 数据是证据，不是数字堆砌。
- AI 面板是研判区，不是聊天窗口。
- 风险信息保持克制，不制造灾难感。
- 页面必须有明确的操作秩序：看见态势、识别风险、查看证据、给出建议、进入协同。

## 2. 推荐画布规格

### 演示大屏

- 尺寸：`1920 x 1080`
- 安全边距：左右 `64px`，上下 `40px`
- 主列网格：`12 columns`
- 列间距：`20px`
- 背景：Deep Graphite `#0B1110`

### 桌面网页

- 设计宽度：`1440px`
- 内容最大宽：`1280px`
- 页面边距：`32px`
- 模块间距：`24px`

### 窄屏演示

- 设计宽度：`390px`
- 主信息只保留：平台标题、地图摘要、核心指标、AI 建议入口。
- 不在移动端展示复杂多栏大屏。

## 3. 首页首屏布局

```text
┌────────────────────────────────────────────────────────────┐
│ Command Header                                             │
├───────────────────────┬────────────────────────────────────┤
│ Hero Narrative         │ National Governance Map             │
│ Title / Subtitle       │ City Nodes / Risk Heat / Data Flow  │
│ CTA                    │                                    │
├───────────────────────┴────────────────────────────────────┤
│ Metric Strip: 城市 / 案例 / AI情报 / 共建机构 / 政策动态       │
└────────────────────────────────────────────────────────────┘
```

### 左侧叙事区

- 宽度：约 `38%`
- 内容：
  - 平台名称
  - 一句能力说明
  - 三个短指标标签
  - 主要操作入口
- 视觉：
  - 不使用营销式大色块。
  - 标题干净、留白充足。
  - 只用 Civic Teal 做关键字强调。

### 右侧地图区

- 宽度：约 `62%`
- 内容：
  - 全国轮廓或城市网络图
  - 已点亮城市节点
  - 待加入城市节点
  - 低饱和风险热区
  - 数据协同线
- 视觉：
  - 地图线条细、低对比。
  - 城市节点有状态差异。
  - 风险色只占局部。

### 底部指标条

- 高度：`104-132px`
- 每个指标一张 Metric Tile。
- 主数字比标签醒目，但不做炫光。

## 4. 地图页布局

```text
┌────────────────────────────────────────────────────────────┐
│ Command Header + Layer Controls                            │
├──────────────┬──────────────────────────────┬──────────────┤
│ Region List  │ National Map                  │ City Detail  │
│ Filters      │ Nodes / Flows / Risk Layer    │ Case / Policy│
├──────────────┴──────────────────────────────┴──────────────┤
│ Event Timeline / Collaboration Feed                         │
└────────────────────────────────────────────────────────────┘
```

### 中心地图

- 占页面最大面积。
- 城市节点和风险热区保持清晰分层。
- 点击城市后，右侧卡片展开，不遮挡地图核心区域。

### 左侧筛选

- 只放高价值筛选：
  - 城市状态
  - 治理主题
  - 风险等级
  - 案例类型
- 不做复杂后台筛选器。

### 右侧城市详情

- 字段：
  - 城市名称
  - 所属省份
  - 已收录案例数
  - 政策动态数
  - 共建机构数
  - 活跃状态
  - 最近动态
- 交互：
  - 提交本城市案例
  - 加入城市共建

## 5. AI 治理助手布局

```text
┌────────────────────────────────────────────────────────────┐
│ Command Header                                             │
├────────────────────────────┬───────────────────────────────┤
│ Intelligence Feed           │ AI Analysis Panel             │
│ Policy / Case / Risk Cards  │ Summary / Evidence / Actions │
├────────────────────────────┴───────────────────────────────┤
│ Related Cases / Policy Evidence / Risk Signals              │
└────────────────────────────────────────────────────────────┘
```

### 左侧信息流

- 像 Bloomberg 信息流：密度高、层级清楚。
- 每条内容必须有来源、地区、发布时间、AI 摘要状态。

### 右侧 AI 研判

- 不出现拟人头像。
- 不使用聊天气泡作为主界面。
- 结构为：
  - 输入来源
  - AI 摘要
  - 关联案例
  - 风险等级
  - 建议动作

## 6. 数据驾驶舱布局

```text
┌────────────────────────────────────────────────────────────┐
│ Command Header                                             │
├─────────────┬───────────────────────────────┬──────────────┤
│ KPI Column  │ Governance Situation Board     │ AI Insight   │
├─────────────┼───────────────────────────────┼──────────────┤
│ Risk Queue  │ Trend Charts / Heat Map        │ Case Signals │
└─────────────┴───────────────────────────────┴──────────────┘
```

### KPI Column

- 社区数量
- 风险事件
- 物业透明度
- 协同效率
- AI 已分析信息

### Situation Board

- 城市治理态势图
- 舆情趋势
- 风险类型分布
- 协同响应时间

### AI Insight

- 今日风险研判
- 重点城市提醒
- 可复用案例建议

## 7. 卡片比例规范

- Metric Tile：`1:0.55`
- Governance Card：`1:0.72`
- Risk Card：`1:0.64`
- AI Analysis Panel：高度随内容，最小 `360px`
- Map Panel：`16:9` 或 `21:10`

## 8. 禁止布局

- 不做满屏同质卡片墙。
- 不做 3D 地球中心构图。
- 不做左右堆满无关指标的传统大屏。
- 不做大面积霓虹边框。
- 不用红色闪烁警报占据视觉中心。
- 不把 AI 助手做成客服聊天窗口。
