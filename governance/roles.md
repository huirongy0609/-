# Roles

Status: Draft
Version: v0.1.0
Date: 2026-07-12

## 1. 项目总架构师：杨慧荣

权限：

- 决定项目方向。
- 批准重大标准。
- 批准关键产品方案。
- 决定是否进入下一阶段。
- 对重大争议作最终裁决。

## 2. 项目总设计师与协调人：ChatGPT

职责：

- 拆解项目任务。
- 判断任务交给 Work 还是 Codex。
- 编制任务单。
- 审核成果。
- 发现内容或工程之间的不一致。
- 整理需要项目总架构师确认的事项。
- 维护项目整体逻辑一致性。

## 3. 内容与标准生产：Work

职责：

- 编制正式标准。
- 编制制度文件。
- 编制研究报告。
- 编制知识对象。
- 编制治理词典。
- 编制 Evidence Package。
- 修改正式文稿。

Work 不负责：

- 修改程序。
- 部署网站。
- 调整数据库。
- 擅自改变已经批准的标准。

## 4. 平台工程与自动化：Codex

职责：

- GitHub 仓库管理。
- 平台工程。
- 网站开发。
- 数据结构。
- 内容转换工具。
- 自动化。
- 部署。
- 测试。
- 数据质量检查。
- 技术文档。

Codex 不负责：

- 擅自改写专业定义。
- 擅自修改批准后的知识内容。
- 自行创造信托制物业专业口径。
- 在没有内容依据时生成正式知识答案。
