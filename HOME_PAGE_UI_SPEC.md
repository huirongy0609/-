# HOME_PAGE_UI_SPEC

本文件定义 `信托制社区治理平台` 首页 UI 规范。

本阶段仅输出设计规范，不涉及开发。

## 1. 首页定位

首页是行业平台门户。

首页不是：

```text
后台首页
数字首页
统计首页
产品商城首页
课程商城首页
软件官网首页
```

首页必须让用户在短时间内理解：

- 平台是谁发起的。
- 平台传播什么理念。
- 平台沉淀哪些内容。
- 平台提供哪些制度资源。
- 平台如何帮助人成长。
- 平台有哪些产品与服务。
- 用户如何进入资源连接。

## 2. 首页必须回答的问题

```text
最近发生了什么
有哪些值得关注的实践
有哪些制度资源
如何进入成长世界
如何连接资源
```

## 3. 整体视觉方向

参考：

- Apple 的留白与克制
- Notion 的内容组织
- Linear 的专业秩序
- Stripe 的产品表达
- 得到的知识平台气质

首页应呈现：

```text
专业
可信
温暖
内容感
成长感
长期主义
```

## 4. 页面结构

建议首页由以下区块组成：

```text
第一屏：品牌总入口
第二屏：最近发生了什么
第三屏：值得关注的实践
第四屏：制度资源
第五屏：金牌管家成长世界
第六屏：产品中心
第七屏：用户识别与资源连接
第八屏：治理实验室
第九屏：AI信托助手
```

## 5. 第一屏：品牌总入口

### 内容

```text
聚道研究院 出品

信托制社区治理平台

传播信托理念，沉淀治理经验，培养共治人才

让更多社区走向公开透明、共建共治与价值共享。
```

### CTA

```text
进入内容中心
了解成长世界
建立我的画像
```

### UI要求

- 以文字和留白建立信任。
- 不使用大图轮播。
- 不使用数字驾驶舱。
- 不使用全国地图主视觉。
- 不使用强发光背景。
- 可使用温和的品牌色细线、轻背景块、内容摘要。

## 6. 第二屏：最近发生了什么

目标：让平台有“持续更新”的真实感。

内容可包括：

```text
最新政策动态
最新实践观察
最新活动报道
最新参访纪实
```

UI形式：

- 时间线式内容流。
- 编辑精选列表。
- 左侧主内容，右侧轻摘要。

禁止：

- 新闻门户式密集列表。
- CMS 栏目堆砌。
- 自动滚动资讯墙。

## 7. 第三屏：值得关注的实践

目标：展示治理经验沉淀。

内容可包括：

```text
实践案例
治理样板
深度观察
人物访谈
```

UI形式：

- 1个重点实践 + 3个辅助案例。
- 每个实践卡片包含标题、城市/角色、摘要、关键词。
- 案例不应像数据库条目，应像编辑精选内容。

## 8. 第四屏：制度资源

目标：让用户看到平台的工具价值。

内容可包括：

```text
法律法规
政策文件
制度模板
工具包
```

UI形式：

- 分类入口。
- 适用对象提示。
- 使用场景提示。

示例信息结构：

```text
制度模板包
适合：物业公司 / 业委会 / 社区
用于：公共收益公开、预算说明、议事规则
```

## 9. 第五屏：金牌管家成长世界

目标：建立成长路径心智。

核心文案：

```text
信托制物业行业成长操作系统
```

必须体现：

- 新手村到五星管家的成长路径。
- 成长档案。
- 勋章系统。
- AI陪练。
- AI考官。

UI形式：

- 横向或纵向成长路径。
- 阶段卡片。
- 当前推荐任务。
- 成就轻提示。

禁止：

- 课程超市。
- 价格卡片。
- 课程封面墙。
- 游戏化过度。

## 10. 第六屏：产品中心

目标：展示产品矩阵，但不商城化。

产品分类：

```text
图书产品
训练营产品
专题课程
软件产品
工具产品
未来产品
```

UI形式：

- 产品矩阵。
- 每类产品说明解决的问题。
- 重点展示“适合谁”和“解决什么”。

禁止：

- 价格优先。
- 促销优先。
- 商品货架式布局。
- 淘宝风、知识店铺风。

## 11. 第七屏：用户识别与资源连接

目标：引导用户建立画像。

核心流程：

```text
先识别用户
再推荐内容
再推荐成长
再推荐产品与资源
最后完成连接
```

首页只展示轻入口：

```text
我是物业从业者
我是社区工作者
我是业委会成员
我是街道/社区负责人
我是治理研究者
我想了解信托制
```

UI形式：

- 温和身份选择。
- 不像注册表单。
- 不强迫登录。
- 说明画像用途。

## 12. 第八屏：治理实验室

目标：保留真实治理样板间。

当前重点：

```text
停车治理样板间
```

展示方式：

- 作为一个可进入的案例样板。
- 说明它用于理解真实治理流程。
- 不作为首页主心智。

## 13. 第九屏：AI信托助手

目标：说明 AI 是全站助手。

能力：

```text
找制度
找内容
找案例
找模板
找课程
生成材料
```

UI形式：

- 小型助手入口。
- 搜索框式入口。
- 内容页和制度页中的辅助按钮。

禁止：

- 独立 AI 中心。
- 聊天机器人占据首页主视觉。
- AI驾驶舱。

## 14. 首页信息层级

优先级：

```text
品牌使命
最近更新
实践案例
制度资源
成长世界
产品服务
用户识别
治理实验室
AI助手
```

## 15. 首页风格禁区

禁止：

- 大数字统计首页。
- 治理态势驾驶舱。
- 全国地图主视觉。
- 发光边框。
- 科技蓝背景。
- 后台侧边栏。
- CMS密集栏目。
- 商城首页。
