# GROWTH_WORLD_UI_SPEC

本文件定义 `金牌管家成长世界` UI 规范。

本阶段仅输出设计规范，不涉及开发。

## 1. 成长世界定位

金牌管家成长世界是：

```text
信托制物业行业成长操作系统
```

它不是：

```text
课程中心
课程超市
产品中心
商城
考试系统
游戏系统
```

它负责：

```text
培养人才
训练人才
评价人才
认证人才
沉淀人才
```

## 2. 视觉参考

参考：

- Duolingo：成长路径、任务感、陪伴感。
- Coursera：学习进度、课程结构、能力目标。
- Notion：成长档案、知识组织。
- Nintendo成长地图：阶段感、探索感、轻成就。

注意：只学习“成长体验”，不学习游戏化过度的视觉。

## 3. 设计目标

用户进入成长世界后，应感受到：

- 自己处在一条清晰成长路径上。
- 平台知道自己当前阶段。
- 每一步都有明确任务。
- 完成任务后有成长记录。
- AI可以陪练、考核和反馈。

## 4. 页面结构

建议结构：

```text
顶部：成长世界定位
个人成长状态：当前等级 / 当前任务 / 最近进度
成长路径：新手村 → 一星 → 二星 → 三星 → 四星 → 五星
今日任务：推荐训练任务
能力模块：公开透明 / 沟通协同 / 制度理解 / 案例复盘 / 风险处理
成长档案：学习记录、任务记录、认证记录
AI陪练与AI考官：情景训练入口
勋章系统：阶段性成果
```

## 5. 成长路径 UI

成长路径阶段：

```text
新手村
一星管家
二星管家
三星管家
四星管家
五星管家
```

UI原则：

- 路径清晰。
- 当前阶段突出。
- 未解锁阶段保持温和提示。
- 已完成阶段有低调成就感。
- 不做夸张游戏地图。

## 6. 阶段卡片

阶段卡片字段：

```text
阶段名称
阶段目标
核心能力
推荐任务
完成状态
下一步建议
```

示例：

```text
二星管家
目标：理解公开透明与规则执行
核心能力：公共收益公开 / 预算说明 / 权责边界
当前任务：完成一次公共收益说明模拟
```

## 7. 任务系统 UI

任务不是普通课程。

任务类型：

```text
阅读任务
案例复盘
情景问答
沟通模拟
制度理解
材料生成
线下参与
```

任务卡片字段：

```text
任务名称
任务目标
预计时间
对应能力
完成状态
AI辅助入口
```

## 8. 成长档案 UI

成长档案应像个人成长记录，不像后台档案。

字段：

```text
当前等级
学习进度
完成任务
参与案例
获得勋章
AI反馈
能力短板
下一步建议
```

UI形式：

- 时间记录。
- 能力雷达可以使用，但必须克制。
- 成长建议优先于分数。
- 不做绩效考核面板。

## 9. 勋章系统 UI

勋章应克制、有纪念感。

示例：

```text
透明沟通勋章
公共收益说明勋章
停车治理参与勋章
社区协同勋章
案例复盘勋章
带教贡献勋章
```

原则：

- 不做游戏币。
- 不做炫光徽章。
- 不做排行榜刺激。
- 勋章要对应真实能力和真实任务。

## 10. AI陪练 UI

AI陪练定位：

```text
帮助用户理解知识、拆解任务、模拟沟通。
```

UI形式：

- 情景训练入口。
- 对话可以存在，但不是聊天产品主视觉。
- 每次训练有目标、角色、反馈。

示例：

```text
模拟场景：居民质疑公共收益去向
训练目标：解释信托制公开透明机制
AI角色：居民代表
```

## 11. AI考官 UI

AI考官定位：

```text
用于问答、情景题、案例复盘评分。
```

UI形式：

- 情景题。
- 案例题。
- 复盘题。
- AI反馈。

禁止：

- 刷题软件风。
- 考试系统后台风。
- 大量分数压迫。

## 12. AI NPC UI

AI NPC 用于模拟治理角色。

角色：

```text
居民
业委会成员
物业项目经理
街道工作人员
社区书记
专家顾问
```

UI目标：

- 帮助练习真实治理沟通。
- 增强场景感。
- 不做游戏角色化头像系统。

## 13. 成长世界视觉禁区

禁止：

- 课程商城。
- 商品货架。
- 游戏化过度。
- 排行榜刺激。
- 炫光徽章。
- 复杂考试后台。
- 知识付费销售页。

## 14. 与其他模块关系

成长世界连接：

- 内容中心：阅读材料和案例素材。
- 制度中心：制度模板和工具包。
- 产品中心：训练营、专题课程、图书。
- 用户识别系统：根据画像推荐成长任务。
- AI信托助手：找课程、找案例、生成训练材料。
