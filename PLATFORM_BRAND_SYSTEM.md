# PLATFORM_BRAND_SYSTEM

本文件定义 `信托制社区治理平台` 的品牌系统、产品矩阵与用户价值路径。

本阶段仅输出品牌与产品体系设计，不涉及开发。

## 1. 品牌架构

品牌层级：

```text
聚道研究院
  ↓
信托制社区治理平台
  ↓
金牌管家成长世界
  ↓
产品矩阵
```

### 聚道研究院

职责：

- 研究
- 标准
- 智库
- 内容
- 行业判断
- 理念源头

边界：

- 聚道研究院是上层品牌与专业背书。
- 它负责回答“为什么做信托制社区治理”。
- 它不直接承担平台日常内容分发、用户识别、产品转化或成长任务。

### 信托制社区治理平台

职责：

- 传播信托理念
- 沉淀治理经验
- 组织内容与制度资源
- 连接社区治理参与者
- 承接用户需求
- 引导用户进入成长、产品与资源路径

边界：

- 平台是统一入口。
- 平台负责聚人。
- 平台不是物业管理系统、治理驾驶舱、项目管理系统或 ERP。
- 平台不直接等同于课程中心、商城或软件官网。

### 金牌管家成长世界

职责：

- 培养人才
- 训练人才
- 评价人才
- 认证人才
- 沉淀人才
- 形成信托制物业行业成长路径

边界：

- 成长世界负责育人。
- 成长世界不是课程中心。
- 成长世界不是产品中心。
- 成长世界不是商城。
- 成长世界是信托制物业行业成长操作系统。

### 产品矩阵

职责：

- 承接平台产品展示
- 承接产品销售
- 承接产品服务
- 承接商业化转化

边界：

- 产品矩阵不是平台本身。
- 产品矩阵不是成长世界。
- 产品矩阵为内容、制度、成长与用户需求提供可交付产品。

## 3. 首页价值主张重构

首页不从平台介绍出发。

首页不从功能模块出发。

首页从用户正在面对的治理问题出发。

### 首页核心提问

```text
今天你遇到什么治理问题？
```

辅助说明：

```text
输入你的问题，平台将为你找到相关内容、制度资源、实践案例、成长任务与可连接资源。
```

### 首页问题入口示例

```text
停车治理
公共收益
业主大会
项目亏损
信托制转型
预算公开
业委会协同
维修资金
物业服务争议
社区共治
```

### 首页第一屏价值主张

推荐表达：

```text
遇到社区治理问题，从这里找到方法、制度、案例和同行。
```

备选表达：

```text
把分散的治理经验，变成可学习、可复用、可连接的公共知识。
```

```text
从一个真实问题出发，找到信托制社区治理的解决路径。
```

### 首页第一屏信息结构

```text
聚道研究院 出品

信托制社区治理平台

今天你遇到什么治理问题？

[请输入你的问题……]

快捷问题：
停车治理 / 公共收益 / 业主大会 / 项目亏损 / 信托制转型

平台可为你提供：
相关内容
制度模板
实践案例
成长任务
资源连接
```

### 首页第一屏用户路径

```text
用户看到问题输入框
  ↓
选择或输入治理问题
  ↓
平台引导建立轻量画像
  ↓

```

```text

产品价格
课程货架
后台菜单
AI大模型炫技
```

## 4. 用户价值路径

完整路径：

```text
用户进入平台
  ↓
建立画像
  ↓
发现内容
  ↓
找到制度
  ↓
学习案例
  ↓
进入成长世界
  ↓
连接资源
  ↓
解决问题
```

### Step 1：用户进入平台

用户问题：

```text
我遇到了一个社区治理问题，不知道从哪里开始。
```

平台价值：

- 用问题入口降低理解门槛。
- 让用户不需要先理解平台结构。
- 从真实治理问题进入内容、制度、案例和资源。

### Step 2：建立画像

用户问题：

```text
我是谁？我所在城市和单位是什么？我现在处在什么阶段？
```

平台价值：

- 识别身份、城市、单位、当前状态、关注方向、需求方向。
- 让平台知道用户需要内容、制度、成长、产品还是资源连接。
- 为后续推荐建立基础。

### Step 3：发现内容

用户问题：

```text
有没有人写过、讲过、研究过类似问题？
```

平台价值：

- 推荐政策动态、实践案例、深度观察、人物访谈、参访纪实和活动报道。
- 将分散经验变成可阅读内容。
- 帮助用户建立基本判断。

### Step 4：找到制度

用户问题：

```text
有没有制度依据、模板、工具包可以参考？
```

平台价值：

- 提供法律法规、政策文件、制度模板和工具包。
- 把理念转化为可执行材料。
- 降低用户从认知到行动的成本。

### Step 5：学习案例

用户问题：

```text
类似问题在别的社区是怎么解决的？
```

平台价值：

- 推荐可复用的实践案例。
- 展示问题、方法、协同关系和阶段成果。
- 让用户从真实经验中学习。

### Step 6：进入成长世界

用户问题：

```text
我能不能系统学习并提升处理问题的能力？
```

平台价值：

- 引导用户进入金牌管家成长世界。
- 提供成长路径、训练任务、AI陪练、AI考官和成长档案。
- 从解决单个问题走向能力成长。

### Step 7：连接资源

用户问题：

```text
我需要找人、找机构、找课程、找工具、找服务。
```

平台价值：

- 根据画像与需求推荐资源。
- 连接内容、制度、产品、专家、机构和项目机会。
- 承接案例投稿、项目需求、参访申请、讲课邀请和学习申请。

### Step 8：解决问题

用户问题：

```text
我如何把方法落到自己社区？
```

平台价值：

- 帮助用户形成行动路径。
- 推荐工具包、训练营、顾问服务、项目诊断和城市辅导。
- 把平台价值从“看到内容”延伸到“推动改变”。

## 8. 成长世界定位强化

核心原则：

```text
金牌管家成长世界不是课程中心。
金牌管家成长世界不是商城。
金牌管家成长世界不是产品中心。

金牌管家成长世界是信托制物业行业成长操作系统。

平台负责聚人。
成长世界负责育人。
```

成长世界的核心价值：

- 将信托制理念变成可训练能力。
- 将真实治理案例变成成长任务。
- 将用户学习过程沉淀为成长档案。
- 将优秀实践者沉淀为行业人才。

成长世界与产品中心的关系：

```text
成长世界提供成长路径和人才能力体系。
产品中心提供可购买、可报名、可交付的产品与服务。
```

## 12. 首页品牌表达原则

首页第一眼不应是：

```text
我们是谁
我们有哪些功能
我们有哪些产品
我们有多少数据
```

首页第一眼应是：

```text
你遇到了什么治理问题？
我们帮你找到内容、制度、案例、成长路径和资源连接。
```

首页从用户问题进入，平台能力在解决路径中自然出现。
