# TRUST_WORLD_PRODUCT_SPEC_V1

本文件定义 `Trust World（信托制世界）` 的产品架构。

它基于：

```text
ROLE_SYSTEM_V1
TRUST_WORLD_STRUCTURE_V1
```

本阶段只定义产品逻辑，不开发、不写代码、不设计页面、不新增路由。

## 1. 产品定义

Trust World 不是资讯网站。

Trust World 是：

```text
中国信托制实践入口与资源连接平台。
```

更完整的定义：

```text
Trust World 是面向不同社区治理角色的信托制认知、实践地图、案例制度与资源连接入口。
```

它的核心价值不是“发布内容”，而是：

```text
让用户知道信托制是什么
让用户看到全国谁在实践
让用户进入自己的角色世界
让用户找到案例、制度、工具和资源
让合适的人进入金牌管家成长世界
```

## 2. 首页用户路径设计

### 2.1 完整路径

```text
首次访问
  ↓
认识信托制
  ↓
发展地图
  ↓
角色选择
  ↓
角色世界
  ↓
资源连接
  ↓
成长世界
```

### 2.2 第一次进入平台：第一眼看到什么

第一眼不应是资讯列表，也不应是政策栏目。

第一眼应看到：

```text
信托制正在成为社区治理的新实践。

你可以从这里了解信托制、看到全国实践、找到属于你的角色路径。
```

首屏必须给用户三个入口：

```text
认识信托制
查看中国信托制实践网络
选择我的身份
```

### 2.3 第二眼点击什么

用户第二眼应根据自己的状态选择：

```text
我第一次听说信托制 → 认识信托制
我想知道哪里在做 → 中国信托制实践网络
我想知道和我有什么关系 → 选择我的身份
```

### 2.4 第三步进入哪里

进入路径：

```text
认识信托制 → 信托制基础解释页
中国信托制实践网络 → 地图首页
选择我的身份 → 我的信托制世界
```

### 2.5 第四步如何形成认知

用户通过以下资产形成认知：

```text
基础解释
权威媒体
中国实践网络
真实案例
制度工具
实践人物
```

认知建立顺序：

```text
概念可信
  ↓
实践可信
  ↓
角色相关
  ↓
工具可用
  ↓
资源可连接
```

### 2.6 第五步如何进入成长世界

进入 Growth World 的触发条件：

```text
用户是物业项目经理
用户希望成为信托制人才
用户希望系统学习
用户浏览了案例并希望训练能力
企业负责人希望培养团队
顾问/培训机构希望参与交付
```

入口文案：

```text
进入金牌管家成长世界
开始我的成长路径
成为信托制实践人才
为团队建立成长体系
```

禁止使用：

```text
购买课程
进入商城
立即下单
限时优惠
```

## 3. 首页模块优先级排序

### 3.1 P0 必须存在

P0 模块用于回答 Trust World 最核心问题。

```text
信托制介绍
中国信托制实践网络
角色入口
```

#### 信托制介绍

回答：

```text
什么是信托制
它解决什么问题
它和我有什么关系
```

#### 中国信托制实践网络

回答：

```text
哪里正在实践信托制
有哪些城市、项目、案例和人物
```

#### 角色入口

回答：

```text
我是哪个角色
我应该看什么
我下一步做什么
```

### 3.2 P1 重要

P1 模块用于建立信任和提供可操作资源。

```text
案例库
制度工具库
媒体报道
```

#### 案例库

证明信托制不是概念，而是有真实实践。

#### 制度工具库

帮助用户从理解进入行动。

#### 媒体报道

提供权威背书，降低陌生用户疑虑。

### 3.3 P2 后续扩展

P2 模块用于增强长期内容厚度和专业网络。

```text
人物库
专题研究
数据报告
```

#### 人物库

让用户看到真实实践者。

#### 专题研究

承接研究者、顾问、机构用户。

#### 数据报告

后续用于表达行业趋势、城市实践发展和人才成长数据。

## 4. MAP_PRODUCT_SPEC_V1

### 4.1 地图产品定位

地图名称：

```text
中国信托制实践网络
```

它不是政府地图。

它不是行政区划展示。

它是：

```text
全国信托制实践、案例、城市、人物和项目的可视化入口。
```

### 4.2 地图首页长什么样

地图首页必须传达：

```text
信托制正在从一个城市走向全国。
```

地图首页结构：

```text
顶部说明
中国信托制实践网络

核心视觉
全国城市节点网络

辅助信息
已收录城市
试点项目
实践案例
参与人物
最近更新

入口
点击城市查看城市专区
```

视觉要求：

- 像全国实践网络。
- 不像政务地图。
- 不像数据大屏。
- 不用蓝色科技风。
- 城市节点应体现实践状态，而不只是地理位置。

### 4.3 城市节点状态

城市节点可分为：

```text
已实践
试点中
案例沉淀中
待补充
```

节点展示：

```text
城市名称
实践状态
试点数量
案例数量
最近更新
```

### 4.4 点击城市后长什么样

点击城市后进入城市专区。

城市专区不是城市官网。

它是该城市信托制实践的聚合页。

城市专区结构：

```text
城市概览
  ├─ 城市名称
  ├─ 省份
  ├─ 实践状态
  ├─ 试点数量
  ├─ 案例数量
  └─ 最近更新

城市政策
城市案例
城市人物
城市项目
媒体报道
参访入口
```

### 4.5 城市专区内容

#### 政策

展示该城市或所在省份与信托制、社区治理、物业治理相关的政策。

字段：

```text
标题
发布单位
发布时间
适用范围
政策摘要
相关案例
```

展示该城市实践案例。

```text

核心问题
信托制方法
阶段结果
```

#### 人物

展示该城市的实践人物。

字段：

```text
姓名

单位
实践方向
代表案例
是否可连接
```

#### 项目

展示试点项目、样板项目、参访项目或可连接项目。

字段：

```text
项目名称
项目类型
所在区域
实践状态
参与主体
是否开放参访
```

#### 媒体报道

展示该城市相关报道。

字段：

```text
媒体名称
报道标题
报道时间
报道摘要
相关城市/项目
```

#### 参访入口

用于承接参访申请。

字段：

```text
参访主题
适合对象
参访城市
参访项目
申请方式
```

### 4.6 城市数据字段定义

```text

实践状态

代表项目
```

## 7. 成长世界导流设计

Trust World 中的成长世界导流应发生在用户已经形成基础认知之后。

```text

项目经理查看项目问题内容后
```

## 9. 产品闭环

Trust World 的产品闭环：

```text
用户首次访问
  ↓
认识信托制
  ↓
看到全国实践网络
  ↓
选择自己的角色
  ↓
进入角色世界
  ↓
获得内容、案例、制度、工具
  ↓
通过 Resource Hub 连接资源
  ↓
合适用户进入 Growth World
  ↓
成长、认证、培训、工具、服务转化
```

## 10. 评审问题回答

### 10.1 用户怎么进入？

用户通过三个入口进入：

```text
认识信托制
中国信托制实践网络
选择我的身份
```

### 10.2 进入后先看到什么？

先看到：

```text
信托制是什么
全国哪里在实践
我这个角色应该看什么
```

### 10.3 如何建立信任？

通过：

```text
中国信托制实践网络
城市专区
权威媒体报道
真实案例
人物库
制度工具
```

建立信任。

### 10.4 如何形成转化？

不同角色转化不同：

```text
业主 → 认同信托制
业委会 → 使用制度工具
社区书记 → 申请参访/培训/项目诊断
街道 → 推动试点
物业项目经理 → 进入成长世界
物业企业负责人 → 导入成长体系/软件/顾问服务
创业者 → 获得合作机会
顾问 → 获得客户资源
研究者 → 参与内容共建
```

### 10.5 如何进入成长世界？

当用户出现以下信号时进入：

```text
想成为信托制人才
想系统训练
想获得认证
想提升项目能力
企业想培养团队
顾问想参与培训交付
```

进入方式：

```text
进入金牌管家成长世界
开始我的成长路径
成为信托制实践人才
为团队建立成长体系
```
