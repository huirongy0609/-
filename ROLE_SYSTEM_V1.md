# ROLE_SYSTEM_V1

本文件定义 `信托制生态系统` 的角色系统。

它是 Trust World 与 Growth World 的底层操作系统。

本阶段仅输出规划文档，不开发、不新增页面、不新增路由、不写业务代码、不运行 build。

## 1. 核心判断

平台当前最大缺口不是首页结构，而是角色系统。

现有平台设计已经回答：

```text
平台是什么
有哪些栏目
首页怎么布局
成长世界是什么
产品中心是什么
```

但角色系统必须回答：

```text
谁进入平台
为什么进入平台
他看到什么内容
他需要什么工具
他连接什么资源
他最终转化为什么
```

## 2. 底层路径

用户不是先进入统一的 Trust World。

用户先进入身份识别。

```text
身份识别
  ↓
我的信托制世界
  ↓
角色化内容
  ↓
角色化制度工具
  ↓
角色化资源连接
  ↓
角色化转化目标
```

后续产品路径：

```text
ROLE_SYSTEM_V1
  ↓
Trust World Mobile V1
  ↓
Growth World Mobile V1
```

顺序不得颠倒。

## 5. 普通业主

### 5.1 用户画像

普通业主是小区权益相关方。

他通常不是为了学习系统理论而来，而是因为自己所在小区发生了具体问题。

典型状态：

```text
看不懂账
不信任物业
关心公共收益
对停车收费有疑问
想知道业主有什么权利
第一次听说信托制
```

### 5.2 核心痛点

- 公共收益不透明。
- 物业账目看不懂。
- 业主知情权不清楚。
- 物业和业委会是否可信。
- 不知道如何参与小区治理。

### 5.3 最关心的问题

```text
公共收益去哪了？
账目是否透明？
物业是否可信？
停车收费依据是什么？
业主大会怎么开？
信托制能不能保障业主权益？
```

### 5.4 进入平台后的第一需求

```text
用普通人能听懂的话解释信托制，以及它和自己的权益有什么关系。
```

### 5.5 首页推荐内容

- 信托制是什么。
- 公共收益为什么要公开。
- 业主如何看懂账目。
- 业主视角的真实案例。
- 业主常见问题问答。

### 5.6 推荐制度工具

```text
公共收益说明书
账目公开示例
业主知情权说明
业主大会流程说明
信托制业主手册
停车收费说明模板
```

### 5.7 推荐资源连接

```text
业主公开课
社区说明会
专家答疑
案例参访
业委会沟通资源
信托制说明材料
```

### 5.8 是否进入成长世界

通常不优先。

普通业主的主要路径是认知与认同，不是训练和认证。

### 5.9 商业转化路径

```text
领取业主手册
报名公开课
参加说明会
提交小区问题
转发信托制内容
```

### 5.10 最终转化目标

```text
认同信托制
理解公开透明
愿意参与监督和共建
成为信托制理念传播者
```

## 6. 业委会成员

### 6.1 用户画像

业委会成员是小区公共事务的组织者、监督者和业主代表。

他需要规则、模板、案例和协同方法，帮助自己更稳妥地推动小区治理。

### 6.2 核心痛点

- 程序不熟，容易有风险。
- 公共收益说不清。
- 被业主质疑。
- 与物业、社区关系紧张。
- 缺少可直接使用的制度工具。

### 6.3 最关心的问题

```text
业主大会怎么规范召开？
公共收益如何公开？

如何与社区、物业协同？

如何提升业委会公信力？
```

## 13. 研究者

### 13.1 用户画像

研究者关注信托制社区治理的理论、案例、制度、趋势和公共价值。

- 案例分散。
- 数据不完整。
- 方法论不清晰。
- 缺少长期观察样本。
- 理论和实践之间缺少连接。

## 14. 角色化 Trust World 原则

Trust World 不应是一个统一内容首页。

它应该是：

```text
同一个信托制世界
不同角色入口
不同问题表达
不同内容推荐
不同制度工具
不同资源连接
不同转化目标
```

示例：

```text
业主看到：权益、账目、公共收益、透明、公平。
业委会看到：规则、监督、协同、议事、公共决策。
社区书记看到：基层矛盾、政策依据、社区协同、项目落地。
物业项目经理看到：收费率、投诉、停车治理、业主关系、成长路径。
物业企业负责人看到：转型、人才、软件、项目复制、品牌提升。
创业者看到：行业机会、合作入口、项目需求、服务网络。
顾问看到：客户资源、课程工具、项目诊断、合作交付。
研究者看到：案例、政策、趋势、方法论、长期样本。
```
