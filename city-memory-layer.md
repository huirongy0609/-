# City Memory Layer

运行层：城市记忆系统

目标：让城市拥有治理记忆。

本层不是数据库，也不是日志列表。

它是：

**城市正在记住自己的治理经验。**

## 1. 城市记忆对象

城市记忆包含：

- 历史治理事件
- 风险处理方式
- 案例传播路径
- AI 研判记录
- 协同演化结果
- 政策响应记录
- 风险缓解经验

## 2. 记忆表达方式

### 轨迹残影

用于表达过去发生过的治理路径。

视觉：

- 低透明线条。
- 深灰青色。
- 不动或极慢移动。

### 记忆节点

用于表达已沉淀案例、已完成协同、已处理风险。

视觉：

- 小节点。
- 弱光。
- 可在 hover 或点击时展开。

### 经验传播路径

用于表达某个治理案例从一个城市扩散到另一个城市。

视觉：

- 细线。
- 低透明光点。
- 不做强动画。

## 3. 记忆不是归档

记忆层不是为了回看旧数据。

它的价值是：

- 支持 AI 匹配。
- 支持风险研判。
- 支持城市复用经验。
- 支持形成治理知识网络。

## 4. Mock 记忆样本

第一版可模拟：

- 常州信托制物业案例沉淀。
- 成都停车治理经验复用。
- 广州财务公开协同路径。
- 深圳 AI 社区治理标签体系。

## 5. 验收标准

用户必须感觉：

**城市不是一次性处理事件，而是在积累治理记忆。**
